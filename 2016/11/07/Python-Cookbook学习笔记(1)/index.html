不断学习Python，精通之～第一章

  • 解压序列赋值给多个变量

字符串也是可迭代对象，可以像迭代列表一样迭代字符串

  • 解压可迭代对象赋值给多个变量

first, *middle, last = grades

middle是列表类型

一个使用分割的递归

  • 保留最后N个元素

collections.deque 双端队列

创建：q = deque(maxlen=3) 建立固定大小的队列

方法：append, appendleft, pop, popleft

这里生成器yield，当if条件满足，返回迭代，但是每次队列都会插入元素，无论if条件是否满足，遇到满足pattern的时候，li返回迭代就会有值

  • 查找最大或者最小的N个元素

find_min uses 0.000190 seconds while heapq uses 0.000074 seconds

可以看出来 使用库 比自己写出的函数快了一倍多，原因是这些集合在底层实现，首先会将集合数据进行堆排序后放入一个列表中

如果查找的元素个数相对比较小，使用nlargest(), nsmallest()比较合适，如果查找唯一的话，使用min(), max()比较合适，如果N大小和集合大小相近，则先排序，sorted(items)[:N]

  • 实现一个优先级队列
  • 字典中的键映射多个值

一个键映射多个值，需要将多个值放到另外的容器里，如列表，集合。

使用collections中的defaultdict构造这样的字典，它会自动初始化每个key刚开始对应的值

defaultdict会自动为将要访问的键创建映射实体，另外一个是字典的setdefault方法

d.setdefault('a', []).append(1)

  • 字典排序

一种简单的排序方法

d = sorted(d.items(), key=lambda x: x[1])

按照值进行排序，然后返回一个元组列表，但是字典d本身没有变化

使用OrderedDict

  • 字典的运算
  • 查找两字典的相同点

键keys()支持集合操作，比如集合的并，交，差运算，同样items()方法支持集合操作，但是values()方法不支持，因为值可能不唯一

  • 删除序列相同元素并且保持顺序

值是hashable类型，类似于独一无二的意思

如果值不是可哈希的，需要进行转换

如果直接去除重复元素，那么使用集合set()

  • 命名切片
  • 序列中出现次数最多的元素

collections.Counter类，一个Counter对象就是一个字典，将元素映射到它出现的次数上

Counter对象在制表或者计数的场合是非常有用的，优先选择，而不是手动利用字典实现

  • 通过某个关键字排序一个字典列表
  • 排序不支持原生比较的对象

key可以传入callable对象，这个对象对每个传入的对象返回一个值，这个值会被sorted用来排序这些对象。

str()一般是将数值转成字符串。

repr()是将一个对象转成字符串显示，注意只是显示用，有些对象转成字符串没有直接的意思。如list,dict使用str()是无效的，但使用repr可以，这是为了看它们都有哪些值，为了显示之用。

可以使用lambda函数，也可以使用operator.attrgetter()方法，使用后者能快一点，attrgetter作用于类，itemgetter用于字典类型

也可以用在min，max函数上

总结一下：key可以用 lambda，operator.itemgetter()用于字典，operator.attrgetter()用于类

  • 通过某个字段将记录分组

itertools.groupby()函数扫描整个序列并且查找连续相同的值的元素序列，每次迭代的时候返回一个值和一个迭代器对象，这个迭代器对象可以生成元素值全部等于上面那个值的组中所有对象。但是必须根据指定的字段将数据排序，groupby函数仅仅检查连续的元素。

如果单纯的分组，可以使用defaultdict

  • 过滤序列元素

列表推导式，[n for n in mylist if n < 0]

生成器表达式，占用少量内存：pos = (n for n in mylist if n > 0),for x in pos:

过滤规则比较复杂，则使用filter()函数
