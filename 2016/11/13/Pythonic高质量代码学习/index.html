Pythonic高质量代码学习

Nov 13 2016

Python之禅，编写高质量代码，改善Python程序的91个建议的学习笔记～

写更加Pythonic的代码，比如空格，注释，空行这样合适的代码风格，以及更加Python味

  • 变量交换

a, b = b, a

  • 安全地关闭文件描述符
  • 序列反向

print(a[::-1]) or print list(reversed(a)))

  • 字符串格式化

print('Hello %(name)s!' % {'name': 'Tom'})

使用字典进行格式化，占位符是键名

使用str.format()方法

print('{greet} from {language}.'.format(greet = 'Hello world', language = 'Python'))

最为Pythonic的字符串格式方法

  • 规范

包和模块的命名采用小写，单数形式，短小，包通常仅作为命名空间，如只包含__init__.py文件

尽量使用下划线法命名变量名

  • PEP8

编码风格指南，提出保持代码一致性的细节要求，包括了对代码布局，注释，命名规范等方面的要求

风格检查程序PEP8

空格比制表符更受推荐
逻辑之间使用空行
两个函数之间两个空行
导入之后有两行
列表有多项，使用多行表示
不要混用tab键和空格
  • 安装一个自动排版称为PEP风格的小工具
  1. pip install autopep8 --user -U
  2. Pycharm→File→settings→Tools→Extends Tools→添加
  3. Name: Autopep8
  4. Tool settings:

Output Filters→添加，在对话框中的：Regular expression to match output中输入:$FILE_PATH$\:$LINE$\:$COLUMN$\:.*

在程序中选中代码，右键 extends Tools->autopep8

  • 三元操作符

print X if X < Y else Y

  • 注释

块注释，行注释，文档注释

注释和代码隔开一定的距离，同时块注释之后最好多留几行空白再写代码

给外部可访问的函数和方法添加文档注释，描述方法的功能，并对参数，返回值以及可能发生的异常进行说明

  • 添加空行使得代码布局更加优雅，合理

在一组代码表达完一个完整的思路之后，应该用空白行进行间隔

二元运算符左右两边需要空格
逗号和分号之前不需要空格
函数的默认参数两侧不需要空格
强调前面的操作符时候使用空格
函数名和左括号之间，序列索引操作时候序列名和[]之间不需要空格

  • 编写函数的原则

    设计尽量短小，嵌套层次不要过深
函数申明应该合理，参数不要过多
参数设计应该向下兼容
一个函数只做一件事，有利于代码的复用

  • 常量

通过自定义类来实现常量功能

在Python的内建命名空间支持一部分常量如True，最好把常量集中到一个文件中。然后在其他模块中引用常量的时候这样from constant import const

模块名字也是一个常量对象

  • assert发现问题

Python3中，assert语法是这样的assert(x==y, "not equal")

在函数调用后，当需要确认返回值是否合理的时候可以使用断言
当条件是业务逻辑继续下去的先决条件时候可以使用断言

可以在自己的代码中，尝试使用下断言

  • 数据交换值

x, y = y, x

一般情况下Python表达式的计算顺序是从左到右，但是表达式赋值的时候表达式右边的操作数先于左边的操作数计算

所以先计算右边表达式y，x，现在内存中创建元组，然后计算表达式左边，使用解压缩

  • 延迟计算

在需要执行的时候才计算表达式的值

if x and y or if x or y or表达式为真的尽量在前面，and的话false尽量在前面

比如查看是不是缩写，先去判断最后一个字符是不是点，然后再看集合中有没有值

if w[-1] == '.' and w in abbreviations: pass

生成器表达式，每次需要计算的时候才通过yield产生所需要的元素

  • 枚举

使用collections.namedtuple

使用元组名，和一个字符串元组来创建，_make通过list来创建一个对象

Seasons = namedtuple('Seasons', ('Spring', 'Summer', 'Autumn', 'Winter'))._make(range(4))

或者使用空格分开的字符串来创建namedtuple

Seasons = namedtuple('Seasons', 'Spring Summer Autumn Winter')._make(range(4))

print Seasons.Spring

  • 类型检查

不刻意进行类型检查，而是在出错的情况下通过抛出异常来进行处理

基于内建类型扩展的用户自定义类型，type函数不能准确返回结果

在古典类中，所有类的实例的type值都相等

约束用户的输入类型从而使之与期望的类型一致，使用类型对应的工厂函数，使用工厂函数对类型进行转换，否则可以使用isinstance()函数检测

工厂函数看上去有点像函数，实质上是类，当调用时候，实际上是生成了该类型的一个实例，就像工厂生产货物一样

  • 尽量转换成浮点类型再做除法

Python3中/为浮点除法, 96 / 100 == 0.96，//是整数除法

  • eval

eval("1+1==2")

eval函数将字符串str当成有效的表达式来求值并且返回计算结果，传入的globals参数的字典中缺少__builtins__时候，当前的全局命名空间将作为globals参数输入并且在表达式计算之前被解析，locals参数默认与globals相同，如果两者都省略的话，表达式将在eval调用的环境中进行

安全漏洞 eval("__import__("os").system("dir")

使用os模块的system函数执行一些系统命令

  • 获取序列迭代的索引和值

zip()函数，将索引和值zip到一起

enumerate()函数，仅在需要的时候才会产生一个(index, item)对

对于迭代器，使用next()方法获取下一个迭代元素

而csv模块的reader方法产生的对象，必须使用next()函数才可以

  • is与==

id()函数来查看变量在内存中具体的存储空间

is表示的是对象标志符，用来检查对象的标志符是否一致，比较两个对象在内存中是否拥有同一块内存空间,id(x) == id(y)。==表示的是相等，检验两个对象的值是否相等。

另外Python中有字符串驻留机制，较小的字符串，为了提高性能会保留其值的一个副本，创建新的字符串的时候直接指向该副本。但是长字符串，Python内存就各自创建了对象来表示a1和b1，所以两个对象拥有相同的内容，但是对象标志符不相同，is为False但是==为True

  • 字符编码

使用decode('utf-8')解码，然后再进行编码encode('gbk')，decode是将其它编码对应的字符串解码成Unicode，encode方法将Unicode编码转换成为另一种编码，Unicode作为转换过程中的中间编码

python3中默认的编码是utf-8,sys.getdefaultencoding()->utf-8,可以在源文件中进行编码说明

# coding=utf-8 or -*- coding: utf-8 -*-

  • 构建合理的包层次来管理module

包下面必须有一个__init__.py，使包和普通股目录区分，另外这个文件可以控制模块的导入，会执行__init__.py，可以在该文件中定义__all__变量，控制需要导入的包或者模块，__all__ = ['Module', 'Module2', 'Subpackage']

将关系密切的模块组织成一个包，使得项目结构更加完善和合理，增强代码的可维护性和实用性

包，包涵一个或者一系列的Python文件的文件夹，作用是合理组织代码，便于维护和使用，并避免命名冲突
